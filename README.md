# 2. ğŸ§¬ Gitlet Full Merge Implementation

Implement and debug the `merge` command.
The full specification:

### **Definition:**

`java gitlet.Main merge [branch name]`

### Responsibilities:

1. **Find the Split Point**

   * Latest common ancestor between current branch and target.

2. **Handle Early-Exit Cases**

   * If given branch is ancestor of current â†’ print:
     `Given branch is an ancestor of the current branch.`
   * If current branch is ancestor of given â†’
     print: `Current branch fast-forwarded.`
     and check out.

3. **File State Rules**
   Implement exhaustive behavior:

* Modified in given but not current â†’ checkout + stage
* Modified in current but not given â†’ keep
* Modified in same way â†’ no change
* Absent at split; present only in current â†’ keep
* Absent at split; present only in given â†’ checkout + stage
* Present at split; removed in given â†’ delete
* Present at split; removed in current â†’ keep removed
* **Modified in different ways â†’ conflict**

  * Write:

    ```
    <<<<<<< HEAD
    contents of file in current branch
    =======
    contents of file in given branch
    >>>>>>>
    ```

4. **Commit After Merge**

   * Commit message:
     `Merged [given branch] into [current branch].`
   * Parent order: current branch first, given branch second.

5. **Conflict Output**

   * If conflicts occur:
     `Encountered a merge conflict.`

### Failure Cases:

* Staged additions/removals â†’
  `You have uncommitted changes.`
* Branch does not exist â†’
  `A branch with that name does not exist.`
* Merging branch with itself â†’
  `Cannot merge a branch with itself.`
* Untracked file would be overwritten â†’
  `There is an untracked file in the way; delete it, or add and commit it first.`
